🚀 Getting Started

First Time Setup

When you first access Tracey's Sentinel, you'll need to authenticate:

  1. Create Account — Click "Sign Up" to register as a new user
  2. Provide Details — Enter your user ID, email, role, and organization
  3. Set Password — Choose a secure password (minimum 8 characters)
  4. Login — Return to login and enter your credentials

Demo Access

To quickly test the system, use one of the demo accounts:

Username: officer1 | Password: demo123 Username: analyst1 | Password: demo123 Username: prosecutor1 | Password: demo123

Main Dashboard

After login, you'll see:

  • Key Metrics — Overview of evidence and case statistics
  • Quick Actions — Fast access to main functions
  • Recent Activity — Log of recent operations
  • Navigation Tabs — Access different sections

✨ Feature Overview

📷 Scanner

Scan QR codes to quickly access evidence. Point your camera at any generated QR code to retrieve evidence details instantly.

🔍 Search

Full-text search across all evidence, cases, and custody events. Filter by status, date range, or organization.

📊 Analytics

View detailed analytics on evidence intake, custody actions, endorsements, and case patterns. Export data as JSON or CSV.

💚 Health Check

Real-time system health monitoring. Verify ledger chain integrity and cryptographic controls.

✓ Compliance

Track compliance status across multiple frameworks. Monitor control implementations and audit readiness.

📁 Cases

Manage multiple cases with associated evidence. View case-level audit summaries and compliance status.

🔒 Security Posture

View active cryptographic controls, data locations, and security measures. Verify encryption status.

🎯 Role-Based Access

Actions are controlled by role (Officer, Analyst, Prosecutor, Judge). Endorsement thresholds vary by action type.

🔌 API Reference

Authentication Endpoints

POST /auth/login

Authenticate user and retrieve session token.

POST /auth/signup

Register new user account with role and organization.

Evidence Management

POST /evidence/intake

Register new evidence with file upload and SHA-256 hash computation.

POST /evidence/event

Record custody action (TRANSFER, ANALYSIS, etc.) with optional endorsement.

GET /evidence/{evidence_id}/timeline

Retrieve complete chain-of-custody timeline with all events and endorsements.

POST /evidence/{evidence_id}/verify

Verify evidence integrity (SHA-256) and record verification event.

Reporting & Artifacts

GET /evidence/{evidence_id}/report

Generate court-ready JSON report with complete evidence lifecycle.

GET /evidence/{evidence_id}/bundle

Download ZIP bundle containing report, PDF, ledger excerpt, and manifest.

System Operations

GET /health

System health check with ledger chain validation.

GET /security/posture

Security posture assessment with encryption and signing details.

📚 For complete API documentation, visit: http://127.0.0.1:8000/docs

🔧 Troubleshooting

❌ Login Not Working

  • Verify backend is running at http://127.0.0.1:8000
  • Check username and password are correct
  • Clear browser cache (Ctrl+Shift+Delete)
  • Try opening in incognito/private window
  • Check browser console for errors (F12)

❌ Evidence Upload Failing

  • Ensure file size is reasonable (< 2GB)
  • Verify you have upload permissions (FIELD_OFFICER role required)
  • Check backend is responding: GET /health
  • Try smaller file first to isolate issue

❌ Buttons Not Working

  • Refresh the page (Ctrl+R / Cmd+R)
  • Clear LocalStorage: Open console and run localStorage.clear()
  • Verify you're logged in (check header for user info)
  • Check that your role has permission for that action

❌ Slow Performance

  • Check network connection speed
  • Monitor System Performance (Windows Task Manager)
  • Restart backend server
  • Try accessing from different browser
  • Check system resources (CPU, disk space)

💾 Data Not Persisting

  • Verify database file exists: data/sentinel.db
  • Check file permissions on data directory
  • Ensure disk has free space
  • Review backend logs for errors

📖 Glossary

Evidence

Digital or physical item collected during investigation, registered in the system with metadata and encryption.

Chain of Custody

Immutable record of all individuals who have had access to evidence, custody events, and verification actions.

Custody Event

Any action performed on evidence (INTAKE, TRANSFER, ANALYSIS, STORAGE, COURT_SUBMISSION, etc.) recorded on the ledger.

Endorsement

Cryptographic approval of an action by an authorized organization. Multiple endorsements may be required for critical actions.

Ledger

Append-only transaction log using hash-chain validation. Each entry is cryptographically signed.

Case

Logical grouping of related evidence items, tracked for audit and compliance purposes.

Role

User classification (Officer, Analyst, Prosecutor, Judge) that determines available actions and permissions.

Organization

Entity or agency (KPS, FORENSIC_LAB, ODPP, etc.) associated with evidence handling and endorsements.

SHA-256

Cryptographic hash function used to compute evidence integrity fingerprint. Cannot be forged.

Ed25519

Elliptic curve signature algorithm used to cryptographically sign all ledger events for non-repudiation.
